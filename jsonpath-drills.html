Mental Model — The Tree Every Kubernetes resource is a JSON tree. JSONPath is just a way to navigate that tree and pick leaves. Think of it like a file path:

/ = root of the resource
.metadata.name = go into metadata folder, pick name file
.spec.containers[0].image = go into spec, then containers array, pick first item, then its image
.items[*].metadata.name = go into items array, pick ALL items, then each one's name

That's it. Everything else is just syntax around this idea.
Core JSONPath Patterns Memorize these
-o jsonpath='{.metadata.name}'
Get a single field from a single resource. Dot-walk the tree.
-o jsonpath='{.items[*].metadata.name}'
Get a field from every item in a list (when using get pods etc). [*] = all items.
-o jsonpath='{range .items[*]}{.metadata.name}{"\n"}{end}'
Loop + newline. Puts each result on its own line. {"\n"} = newline, {"\t"} = tab.
-o jsonpath='{.items[*].status.addresses[?(@.type=="InternalIP")].address}'
Filter. ?(@.field=="value") selects items where field matches. Like SQL WHERE.
-o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.status.podIP}{"\n"}{end}'
Multi-column output. Print name + tab + IP + newline per item. Build custom tables.
Custom Columns Faster for tables
-o custom-columns=NAME:.metadata.name,IMAGE:.spec.containers[0].image
Print a table with headers. Faster to type than jsonpath range loops. Each column = HEADER:JSONPATH.
-o custom-columns=NODE:.spec.nodeName,STATUS:.status.phase
Works for any fields. No quotes needed around header names. Commas between columns, colons between name:path.
Bash Filtering When JSONPath isn't enough
| grep -i "pattern"
Filter lines containing a pattern. -i = case insensitive. -v = invert (exclude).
| awk '{print $1}'
Print the Nth column. $1 = first, $2 = second, $NF = last. Splits on whitespace.
| wc -l
Count lines. Use with --no-headers to count resources. wc -w counts words.
| sort | uniq -c | sort -rn
Count unique values and sort by frequency. Great for "which node has the most pods?"
| tr " " "\n"
Replace spaces with newlines. Useful when jsonpath outputs space-separated values and you want one-per-line.
Exam Decision Tree Need a single field? → -o jsonpath='{.field}'
Need a list of one field? → -o jsonpath='{.items[*].field}' then pipe to tr " " "\n"
Need a filtered list? → -o jsonpath with ?(@.field=="value")
Need a table of multiple fields? → -o custom-columns=A:.path,B:.path
Need to count things? → --no-headers | wc -l
Need to sort? → --sort-by=.path (kubectl native) or pipe to sort
Don't remember the path? → kubectl get pod X -o json | less then browse the tree
The Emergency Fallback When stuck
kubectl get pod X -o json | less
Dump the full JSON. Browse it with arrow keys. /search to find fields. Copy the path you need. This is faster than guessing jsonpath syntax. Use this in the exam — it's allowed.
How to practice Type the FULL kubectl command — not just the jsonpath part. You need to type the whole thing fast in the exam. Wrong is fine — the answer will show you the right pattern. Do each drill 3x on different days and you'll have it memorized.
