Break & Fix Labs

40 hands-on scenarios covering all CKA exam domains. Each lab breaks a real cluster failure; you diagnose and fix it using only kubectl and ssh.

Lab 01 of 40
Node NotReady

Worker node shows NotReady. Fix it.

Mental model Node is like a worker who clocked out — kubelet is the time card. No heartbeat, no work.
What breaks
A worker node stops heartbeating to the control plane. kubelet service stops running on the node.
How it manifests
kubectl get nodes shows NotReady. Pods on that node are evicted to other nodes.
bash — diagnose
copy
kubectl get nodes
kubectl describe node <node-name>
ssh <node-ip>
sudo systemctl status kubelet
sudo systemctl restart kubelet

Lab 09 of 40
etcd Restore

Critical namespace was deleted. Recover it.

Mental model etcd is the cluster's memory. Restore is like loading a brain backup—everything snaps back to that point in time.
What breaks
Data was deleted from etcd (e.g., namespace deleted). etcd currently has the wrong state.
How it manifests
Namespace or resources gone. etcd is running fine. You have an etcd snapshot from before the deletion.
bash — diagnose
copy
ssh <control-plane>
sudo systemctl stop kubelet
sudo mv /etc/kubernetes/manifests/kube-apiserver.yaml /tmp/
sudo etcdctl snapshot restore /path/to/snapshot.db \
  --data-dir=/var/lib/etcd-restore
sudo rm -rf /var/lib/etcd && mv /var/lib/etcd-restore /var/lib/etcd
sudo mv /tmp/kube-apiserver.yaml /etc/kubernetes/manifests/
sudo systemctl start kubelet

Lab 19 of 40
Audit Logging

API server is not recording audit events. Fix it.

Mental model Audit log is the security camera. No camera = no recording of who did what. Enable it on the API server, define the policy, verify the tape rolls.
What breaks
API server audit logging not configured. --audit-policy-file and --audit-log-path flags missing from apiserver manifest.
How it manifests
No audit log file exists. API requests aren't recorded. Compliance and forensics requirements unmet.
bash — fix
copy
ssh <control-plane>
# create audit policy
cat > /etc/kubernetes/audit-policy.yaml <<EOF
apiVersion: audit.k8s.io/v1
kind: Policy
rules:
- level: Metadata
EOF
# add to kube-apiserver manifest:
# --audit-policy-file=/etc/kubernetes/audit-policy.yaml
# --audit-log-path=/var/log/kubernetes/audit.log
sudo vi /etc/kubernetes/manifests/kube-apiserver.yaml

RBAC Design

ServiceAccount is getting Forbidden errors. Fix it.

Mental model ClusterRole = a global access pass. RoleBinding = handing that pass to a specific person. Both must exist and point to each other.

Cross-Namespace RBAC

ServiceAccount gets 403 Forbidden across namespaces. Fix it.

Mental model ClusterRoleBinding is a master key that works in every namespace. RoleBinding only opens one door in one namespace.
What breaks
ServiceAccount in namespace A needs to access resources in namespace B. RoleBinding is missing or scoped incorrectly.
How it manifests
403 Forbidden when ServiceAccount tries to access cross-namespace resources. Auth can-i fails from the source namespace.
bash — fix
copy
kubectl auth can-i list pods --as=system:serviceaccount:<ns>:<sa> -n <target-ns>
kubectl create clusterrolebinding <name> \
  --clusterrole=<role> \
  --serviceaccount=<source-ns>:<sa-name>
kubectl auth can-i list pods --as=system:serviceaccount:<ns>:<sa> -n <target-ns>

etcd Backup

No etcd backup exists. Create one.

Mental model etcd snapshot is a database export. No backup = no recovery. Run it before every change. Store it somewhere safe.
What breaks

How it manifests

bash — backup
copy

  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key

Namespace Isolation

Namespaces can freely reach each other. Isolate them.

Mental model Default Kubernetes = open office, everyone hears everything. NetworkPolicy = glass walls. Add them to isolate tenants from each other.
What breaks
Namespace isolation policy is missing. All namespaces can communicate with each other freely.
How it manifests

yaml — policy
copy

kind: NetworkPolicy
metadata:
  name: deny-cross-namespace
  namespace: <target-namespace>
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector: {}  # only same namespace

Lab 33 of 40
Resource Limits

Pods have no resource limits. Fix it.

Mental model Requests = the seat reservation. Limits = the hard cap. No limits = one pod can eat the whole buffet and starve every other guest.
What breaks
Resource limits not set. Pods can consume unlimited CPU/memory. No LimitRange enforcing defaults.
How it manifests
Node resource pressure. Other pods OOMKilled or CPU throttled. kubectl top pods shows runaway consumption.
bash — fix
copy
kubectl top pods
kubectl describe limitrange  # if exists
kubectl create -f - <<EOF
apiVersion: v1
kind: LimitRange
metadata:
  name: default-limits
spec:
  limits:
  - type: Container
    default: {cpu: 200m, memory: 128Mi}
    defaultRequest: {cpu: 100m, memory: 64Mi}
EOF

Priority & Preemption

Critical pods are stuck behind low-priority workloads. Fix it.

Mental model PriorityClass is first class vs coach. High-priority pod gets a seat even if the flight is full—lower-priority pod gets bumped.
What breaks
PriorityClass not created. Critical pods compete equally with batch jobs. No preemption occurs when cluster is full.
How it manifests
Critical workload stuck in Pending while low-priority pods run. No preemption events in scheduler logs.
bash — fix
copy
kubectl get priorityclass

apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: high-priority
value: 1000000
globalDefault: false
EOF
kubectl patch deployment <name> -p \
  '{"spec":{"template":{"spec":{"priorityClassName":"high-priority"}}}}'

Lab 40 of 40
PVC Expansion

PVC resize request is failing. Fix it.

Mental model Expanding a PVC is like adding lanes to a highway. The StorageClass must explicitly allow it. Without permission, the request is silently ignored.
What breaks
PVC resize requested but StorageClass has allowVolumeExpansion: false (or missing). Resize never applies.
How it manifests
PVC storage size stays unchanged after edit. Events show "volume expansion not permitted." Disk usage maxes out.
bash — fix
copy
kubectl get storageclass
kubectl patch sc <name> \
  -p '{"allowVolumeExpansion":true}'
kubectl edit pvc <pvc-name>
# change spec.resources.requests.storage to new size
kubectl get pvc <pvc-name>  # verify new capacity
